# [Chrome 확장프로그램] 근로복지공단 업무 자동화 만들기 - 1편: 프로젝트 소개

## 들어가며

노무사 사무실에서 일하다 보면 매달 반복되는 업무가 있습니다. 바로 **근로복지공단 토탈서비스**에서 여러 사업장의 근로자 정보를 조회하고 다운로드하는 일이죠.

사업장이 10개, 20개가 넘어가면... 하나씩 클릭하고, 조회하고, 엑셀 다운로드 받고... 이걸 반복하다 보면 30분~1시간이 훌쩍 지나갑니다.

> "이거 자동화하면 안 되나?"

그래서 만들었습니다. **Chrome 확장프로그램**으로!

## 완성된 기능 미리보기

[스크린샷: 확장프로그램 팝업 UI 전체 화면]

### 주요 기능

**1. 일괄 조회 & 다운로드**
- 여러 사업장 정보를 CSV로 입력
- 버튼 한 번으로 전체 사업장 근로자 정보 다운로드
- 진행률 실시간 표시

**2. 맞춤 CSV 출력**
- 필요한 컬럼만 추출 (이름, 주민번호, 월평균보수, 취득일)
- 한글 헤더
- 주민번호 포맷 (000000-0000000)
- 금액 쉼표 표시 (1,800,000)

**3. 다중 사무대행기관 지원**
- 여러 사무대행기관의 사업장도 처리 가능

## 기술 스택

- **Chrome Extension Manifest V3** (최신 버전)
- **Vanilla JavaScript** (프레임워크 없음)
- **Chrome APIs**: cookies, scripting, storage
- **근로복지공단 API** 활용 (리버스 엔지니어링)

---

## 왜 Chrome 확장프로그램인가?

### 다른 방법들의 한계

1. **매크로/RPA 도구**: 화면 기반이라 불안정, 사이트 변경에 취약
2. **외부 서버에서 API 호출**: 인증(공인인증서) 문제로 불가능
3. **Selenium/Puppeteer**: 브라우저 자동화지만 인증서 로그인 처리 어려움

### Chrome 확장프로그램의 장점

- **로그인 세션 공유**: 이미 로그인된 브라우저에서 실행
- **API 직접 호출**: 화면 조작 없이 빠르고 안정적
- **쿠키 접근 가능**: HttpOnly 쿠키도 chrome.cookies API로 읽기 가능
- **설치 간편**: 개발자 모드로 바로 테스트 가능


## 주의사항

이 확장프로그램은 **개인 업무 효율화** 목적으로 제작되었습니다.

- 근로복지공단 서버에 과도한 부하를 주지 않도록 요청 간격(2초)을 두었습니다
- 타인의 계정이나 정보에 접근하는 용도로 사용하면 안 됩니다
- API 구조는 근로복지공단에서 변경할 수 있으며, 그 경우 수정이 필요합니다
