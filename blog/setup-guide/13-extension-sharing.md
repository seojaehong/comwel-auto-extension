# [Claude Code 완벽 가이드] 13편: Chrome 확장프로그램 배포/공유

## 이전 편 요약

12편에서는 bkit 에이전트를 배웠습니다. 이번 편에서는 만든 **확장프로그램을 다른 사람과 공유**하는 방법을 알아봅니다.

---

## 공유 방법 3가지

| 방법 | 난이도 | 비용 | 대상 |
|------|--------|------|------|
| ZIP 파일 공유 | ⭐ | 무료 | 소수 사용자 |
| GitHub 배포 | ⭐⭐ | 무료 | 개발자/기술 사용자 |
| Chrome 웹 스토어 | ⭐⭐⭐ | $5 (1회) | 일반 대중 |

---

## 방법 1: ZIP 파일로 공유 (가장 쉬움)

### 개발자 (보내는 사람)

#### 1. 불필요한 파일 제거

```
확장프로그램 폴더에서 제거할 것:
- .git/ (버전 관리 폴더)
- node_modules/ (있다면)
- .DS_Store (Mac)
- Thumbs.db (Windows)
```

#### 2. ZIP 압축

```
폴더 우클릭 → 압축 → my-extension.zip
```

#### 3. 전달

- 이메일 첨부
- 카카오톡/슬랙
- 구글 드라이브/네이버 클라우드

### 사용자 (받는 사람)

#### 1. ZIP 압축 해제

다운로드한 ZIP 파일을 원하는 위치에 압축 해제

```
예: C:\Users\홍길동\Documents\my-extension\
```

#### 2. Chrome 확장프로그램 페이지 열기

주소창에 입력:
```
chrome://extensions
```

또는:
```
Chrome 메뉴(⋮) → 확장 프로그램 → 확장 프로그램 관리
```

#### 3. 개발자 모드 활성화

![개발자 모드](placeholder-developer-mode.png)

오른쪽 상단의 **"개발자 모드"** 스위치를 켭니다.

#### 4. 압축해제된 확장 프로그램 로드

![압축해제된 확장 프로그램 로드](placeholder-load-unpacked.png)

1. **"압축해제된 확장 프로그램을 로드합니다"** 클릭
2. 압축 해제한 폴더 선택
3. **"폴더 선택"** 클릭

#### 5. 확인

확장프로그램 목록에 추가되면 완료!

---

## 방법 2: GitHub로 공유

### 장점

- 버전 관리 가능
- 업데이트 배포 쉬움
- 오픈소스 프로젝트로 공개

### 개발자 (배포)

#### 1. GitHub 저장소 생성

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/my-extension.git
git push -u origin main
```

#### 2. README.md 작성

```markdown
# 근로복지공단 자동화 확장프로그램

## 설치 방법

1. 이 저장소를 다운로드 (Code → Download ZIP)
2. 압축 해제
3. Chrome에서 `chrome://extensions` 열기
4. 개발자 모드 ON
5. "압축해제된 확장 프로그램을 로드합니다" 클릭
6. 압축 해제한 폴더 선택

## 사용 방법

1. 근로복지공단 사이트 로그인
2. 확장프로그램 아이콘 클릭
3. 원하는 기능 실행
```

### 사용자 (설치)

#### 1. GitHub에서 다운로드

![GitHub 다운로드](placeholder-github-download.png)

1. GitHub 저장소 접속
2. **Code** 버튼 클릭
3. **Download ZIP** 클릭

#### 2. 이후는 ZIP 설치와 동일

압축 해제 → chrome://extensions → 개발자 모드 → 로드

---

## 방법 3: Chrome 웹 스토어 등록

### 장점

- 일반 사용자도 쉽게 설치
- 자동 업데이트
- 공식 배포 채널

### 비용

**개발자 등록비: $5 (1회, 평생)**

### 과정 개요

#### 1. 개발자 계정 등록

1. [Chrome 웹 스토어 개발자 대시보드](https://chrome.google.com/webstore/developer/dashboard) 접속
2. Google 계정으로 로그인
3. $5 결제

#### 2. 확장프로그램 패키징

폴더를 ZIP으로 압축 (manifest.json이 루트에 있어야 함)

```
my-extension.zip
├── manifest.json    ← 반드시 루트에
├── popup/
├── content/
└── ...
```

#### 3. 스토어에 업로드

1. **"새 항목"** 클릭
2. ZIP 파일 업로드
3. 정보 입력:
   - 이름
   - 설명
   - 아이콘 (128x128)
   - 스크린샷 (1280x800 또는 640x400)
   - 카테고리

#### 4. 심사 제출

- 심사 기간: 1~7일
- 정책 위반 시 거부될 수 있음

#### 5. 승인 후 배포

승인되면 스토어에서 검색/설치 가능!

---

## 주의사항

### ZIP/GitHub 공유 시

```
⚠️ 주의: 개발자 모드 확장프로그램 경고

Chrome이 시작할 때마다 경고 메시지가 표시될 수 있습니다:
"개발자 모드 확장 프로그램을 사용 중지하세요"

이건 정상이며, "X" 클릭해서 닫으면 됩니다.
웹 스토어 등록 시 이 경고가 사라집니다.
```

### 보안 관련

```
✅ 공유해도 되는 것:
- 소스 코드 파일
- manifest.json
- 아이콘/이미지

❌ 공유하면 안 되는 것:
- API 키 (하드코딩된 경우)
- 개인정보가 포함된 설정 파일
- .env 파일
```

### 업데이트 배포

| 방법 | 업데이트 방식 |
|------|--------------|
| ZIP | 새 ZIP 전달, 기존 삭제 후 재설치 |
| GitHub | 새 버전 다운로드, 기존 삭제 후 재설치 |
| 웹 스토어 | 자동 업데이트 |
