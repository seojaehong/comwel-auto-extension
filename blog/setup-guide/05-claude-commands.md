## 이전 편 요약

4편에서는 터미널 기본 사용법을 배웠습니다. 이번 편부터 Claude Code를 본격적으로 활용합니다.

## Claude Code 시작하기

### 실행

프로젝트 폴더에서:
```bash
claude
```

### 종료

```
/exit
```
또는 `Ctrl + C` 두 번

---

## 슬래시 명령어 (/)

Claude Code에는 특수 명령어가 있습니다. `/`로 시작합니다.

### 필수 명령어

| 명령어 | 설명 |
|--------|------|
| `/help` | 도움말 보기 |
| `/clear` | 대화 기록 지우기 |
| `/exit` | Claude Code 종료 |
| `/compact` | 대화 요약 (컨텍스트 절약) |

### 자주 쓰는 명령어

| 명령어 | 설명 |
|--------|------|
| `/init` | 프로젝트 초기화 (CLAUDE.md 생성) |
| `/cost` | 현재 세션 비용 확인 |
| `/model` | AI 모델 변경 |
| `/status` | 현재 상태 확인 |

## 대화 방식

### 자연어로 요청

Claude Code는 자연어를 이해합니다. 편하게 말하면 됩니다.

```
이 프로젝트 구조 설명해줘
```

```
package.json에 있는 의존성 패키지들이 뭔지 알려줘
```

```
이 폴더에 index.html 파일 만들어줘
```

### 구체적으로 요청하기

더 좋은 결과를 얻으려면 구체적으로 요청하세요.

**❌ 모호한 요청:**
```
웹사이트 만들어줘
```

**✅ 구체적인 요청:**
```
HTML과 CSS로 간단한 자기소개 웹페이지 만들어줘.
- 이름, 직업, 연락처 섹션
- 반응형 디자인
- 파란색 테마
```

## 명령어 실행

Claude에게 터미널 명령어 실행을 요청할 수 있습니다.

```
git status 실행해줘
```

```
npm install axios 해줘
```

```
이 프로젝트 테스트 실행해줘
```

### 권한 확인

명령어를 실행하기 전에 Claude가 확인을 요청합니다:

```
Claude: 다음 명령어를 실행하시겠습니까?
> npm install axios

[Y/n]
```

**Y** 또는 **Enter**로 승인, **n**으로 거부

## 컨텍스트 관리

### 컨텍스트란?

Claude가 기억하고 있는 대화 내용과 파일 정보입니다.

대화가 길어지면 컨텍스트가 커지고 비용이 증가합니다.

### /compact 사용

대화가 길어지면 요약해서 컨텍스트를 줄일 수 있습니다:

```
/compact
```

### /clear 사용

새로운 주제로 넘어갈 때:

```
/clear
```

대화 기록이 모두 지워지고 새로 시작합니다.

## 효과적인 대화 팁

### 1. 한 번에 하나씩

**❌:**
```
파일 만들고, 테스트하고, 배포해줘
```

**✅:**
```
먼저 파일 만들어줘
```
(완료 후)
```
이제 테스트 실행해줘
```

### 2. 오류 공유하기

오류가 발생하면 **전체 오류 메시지**를 공유하세요.

```
이런 오류가 났어:
Error: Cannot find module 'express'
    at Function.Module._resolveFilename
    ...
```

### 3. 결과 확인하기

Claude가 작업을 완료하면 결과를 확인하세요.

```
만든 파일 내용 보여줘
```

```
실행해서 잘 되는지 확인해봐
```
